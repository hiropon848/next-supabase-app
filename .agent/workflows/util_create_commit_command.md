> [!CRITICAL]
> **実行禁止命令**
> このワークフローの目的は「コマンドの作成と提示」のみである。
> いかなる理由があっても、作成した `git add` や `git commit` コマンドを **実行(`run_command`)してはならない**。
> ユーザーからの明示的な「実行承認」待機状態で終了すること。

1.  **状況確認**: `git status` を実行し、変更されたファイルを確認する。
2.  **内容分析**: 必要であれば `git diff` などで変更点を確認し、変更内容を的確に表すコミットメッセージ（日本語）を考案する。
3.  **コマンド作成**: 確認したファイルをステージングする `git add` 文字列と、考案したメッセージを含む `git commit` 文字列を作成する。
4.  **提示と停止**:
    *   作成したコマンドをコードブロックで提示する。
    *   **「コマンドを作成しました。実行してよろしいですか？」と問いかけ、処理を完全に停止する。**
